## 🎯 修改原因
ai_vanvan 账号因频繁下载被 Instagram 封禁，需要调整下载策略以保护剩余账号。

---

## ⚙️ 配置修改详情

### ai_vanvan 账号配置

#### 修改前（危险配置）⚠️
```json
"download_safety": {
  "max_posts_per_session": 50,   // 单次最多50个
  "request_delay": 2               // 延迟2秒
}
```

**问题：**
- 单次下载量太大（50个）
- 延迟太短（2秒）
- 导致被识别为机器人

---

#### 修改后（安全配置）✅
```json
"download_safety": {
  "max_posts_per_session": 5,    // 单次最多5个
  "request_delay": 45              // 延迟45秒
}
```

**改进：**
- ✅ 单次下载量降低 90%（50 → 5）
- ✅ 延迟增加 22.5倍（2秒 → 45秒）
- ✅ 添加随机延迟（±30%）
- ✅ 更接近人类行为

## 🔧 代码层面改进

### 1. 添加随机延迟机制

**修改文件：** `src/platforms/instagram/downloader.py`

**新增导入：**
```python
import time
import random
```

**修改延迟逻辑：**
```python
# 旧代码（固定延迟）
time.sleep(request_delay)  # 固定2秒

# 新代码（随机延迟）
random_delay = request_delay * random.uniform(0.7, 1.3)
self.logger.info(f"等待 {random_delay:.1f} 秒（安全延迟）...")
time.sleep(random_delay)
```

**效果：**
- 配置延迟 45 秒
- 实际延迟：31.5 - 58.5 秒（随机）
- 模拟人类的不规律行为

## ⏰ 推荐的使用策略

### 下载频率建议

| 时间间隔 | 操作 | 说明 |
|---------|------|------|
| **每天** | ❌ 不推荐 | 太频繁，易触发风控 |

### 每次下载数量

| 数量 | 风险 | 建议 |
|------|------|------|
| 1-3个 | 🟢 安全 | ✅ 推荐 |

| 6-10个 | 🟠 较高 | ❌ 不建议 |
| 10+个 | 🔴 极高 | ❌❌ 禁止 |

**❌ 不好的做法：**
```
固定时间运行（如每天21:30）
→ Instagram 容易识别模式
```

**✅ 好的做法：**
```

明天：不运行
后天：晚上 20:47
大后天：不运行
第5天：早上 09:15
```
