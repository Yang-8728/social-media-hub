## 与旧架构对比

| 特性 | 旧架构 (main.py) | 新架构 (容器化) | 优势 |
|------|-----------------|---------------|------|
| **部署方式** | 单进程运行 | 多容器运行 | 更稳定 |
| **服务启动** | `python main.py` | `docker-compose up` | 一键启动 |
| **依赖管理** | 本地 Python 环境 | 容器内隔离 | 避免冲突 |
| **扩展性** | 单线程处理 | 多服务并行 | 更高效 |
| **监控** | 需要手动查看日志 | `docker logs` | 更方便 |
| **错误恢复** | 整体崩溃 | 单服务重启 | 更可靠 |
